usage: fleet [-h] [--log-level {DEBUG,INFO,WARNING,ERROR}] [--hosts H,H,…]
             [--discover-timeout SEC] [--board CLASS] [--fields F,F,…]
             [--gate NAME] [--skip NAME] [--out-json PATH] [--out-junit PATH]
             [--baseline PATH] [--criteria PATH] [--settle [SEC]] [--dry-run]
             [--yes] [--metrics-mqtt-url HOST[:PORT]] [--metrics-topic PREFIX]
             [--no-publish-metrics]
             SUBCOMMAND ...

TaipanMiner fleet test harness

positional arguments:
  SUBCOMMAND
    discover            discover devices via mDNS and print table
    status              GET /api/info + /api/health per device
    probe-endpoints     spec-driven endpoint crash probe: hit each GET once
                        and flag uptime regressions
    functional          Validate each device's REST API against its own
                        OpenAPI spec
    soak                Long-duration fleet health soak (heap, reboot, WDT,
                        hashrate, publisher)
    stress              Browser-like concurrent load test with crash detection
                        and heap-recovery assertion
    faults              Recovery-asserting fault injection (socket exhaustion,
                        broker outage, vcore drop)
    telemetry           Telemetry transport — configure each telemetry
                        transport and verify publish health
    describe            inspect the served OpenAPI spec
    call                make an arbitrary API request (safety-gated for
                        mutating methods)
    watch               poll an endpoint per-device and observe field values
                        (CI-safe, read-only)
    logs                retrieve device kernel log via GET /api/logs (SSE,
                        read-only)
    ota                 OTA firmware operations
    decode              decode a panic backtrace from a live device (uses
                        archived ELF)
    elf                 ELF archive management

options:
  -h, --help            show this help message and exit
  --log-level {DEBUG,INFO,WARNING,ERROR}
                        log level (default: WARNING)

targeting:
  --hosts H,H,…         comma-separated IPs/hostnames (skip mDNS discovery)
  --discover-timeout SEC
                        mDNS browse window in seconds (default: 10)
  --board CLASS         filter devices by board class substring (e.g. bitaxe)

output:
  --fields F,F,…        comma-separated field subset for on-demand validation
  --gate NAME           enable a specific check gate (repeatable; default:
                        all)
  --skip NAME           disable a specific check gate (repeatable)
  --out-json PATH       write JSON results to file
  --out-junit PATH      write JUnit XML results to file
  --baseline PATH       compare results against a prior JSON result file
  --criteria PATH       YAML criteria override file

run control:
  --settle [SEC]        opt-in warmup settle: bare = criteria default (120s),
                        --settle N = N seconds
  --dry-run             log mutating operations but don't execute them
  --yes                 auto-confirm guard for mutating operations

metrics publishing:
  --metrics-mqtt-url HOST[:PORT]
                        broker for run metrics (overrides
                        BB_TEST_METRICS_BROKER / BB_TEST_RECEIVER); default ON
                        when configured, skip with note when absent
  --metrics-topic PREFIX
                        MQTT topic prefix for run metrics (default:
                        fleettest); full topic: <prefix>/<suite>/<board>
  --no-publish-metrics  disable automatic metrics publishing (opt-out)
